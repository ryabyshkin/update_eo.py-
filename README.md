# 📊 Автообновление ЕО из отчётов 1С

## Структура репозитория

```
repo/
├── .github/
│   └── workflows/
│       └── build_eo.yml          ← GitHub Actions (этот файл)
├── data/
│   ├── EO_template.xlsx          ← Шаблон ЕО (не меняется вручную)
│   ├── vypusk_shi.xlsx           ← Выпуск ШИ из 1С
│   ├── opt.xlsx                  ← ОПТ из 1С
│   ├── ostatok.xlsx              ← Остатки из 1С
│   ├── prodazhi_nedelya.xlsx     ← Продажи за последнюю неделю из 1С
│   ├── rezerv_lamoda.xlsx        ← Резерв Ламода из 1С
│   └── rezervy_obsh.xlsx         ← Резервы общие из 1С
├── output/
│   └── EO_updated.xlsx           ← Результат (генерируется автоматически)
├── update_eo.py                  ← Основной скрипт
└── README.md
```

## Как обновить данные каждую неделю

### Шаг 1 — Выгрузить 6 файлов из 1С и загрузить в папку `data/`

Переименовать файлы ТОЧНО так:

| Файл из 1С            | Название в репозитории      |
|-----------------------|-----------------------------|
| Выпуск ШИ             | `vypusk_shi.xlsx`           |
| ОПТ                   | `opt.xlsx`                  |
| Остатки               | `ostatok.xlsx`              |
| Продажи за неделю     | `prodazhi_nedelya.xlsx`     |
| Резерв Ламода         | `rezerv_lamoda.xlsx`        |
| Резервы общ.          | `rezervy_obsh.xlsx`         |

### Шаг 2 — Указать название новой недели продаж

В файле `update_eo.py` на строке ~47:

```python
NEW_SALES_WEEK_LABEL = "Продажи 09.04.-15.04. шт"
# ↑ Меняй это значение каждую неделю
```

### Шаг 3 — Запушить файлы в репозиторий

```bash
git add data/
git commit -m "Обновление данных 14.04.2026"
git push
```

GitHub Actions автоматически:
1. Запустит скрипт
2. Обновит `output/EO_updated.xlsx`
3. Закоммитит результат обратно

Скачать готовый файл можно из папки `output/` на GitHub.

## Логика ключа связи

Все источники связываются с ЕО по формуле:

```
Номенклатура + " (" + Характеристика_до_точки_с_запятой + ")"
```

Пример:
```
"Бермуды джинсовые (100% хлопок Молочный; Размер: M"  →  "Бермуды джинсовые (100% хлопок Молочный)"
```

## Примечания

- **Поступление фабрика** — заполняется вручную, скрипт не трогает
- **Остатки LA, Продажи LA, Резерв ЕКБ, Продажи ЕКБ** — отдельные файлы (можно добавить аналогично)
- Если значение не найдено в источнике — ячейка остаётся пустой (не "нет данных")
